## Critical Bug Found Post-Run

`load_checkpoint` in `train.py` restores `lambda_val` from the checkpoint metadata:

```python
self.lambda_val = ckpt.get("lambda_val", self.lambda_val)
```

The resume checkpoint (`5d285a` best.pt) had `lambda_val=0.05`. This overwrote the CLI `--lambda 60`. **The entire `85e9d5` run trained with λ=0.05** — the same loss imbalance as the first failed run.

**Note:** The `lambda_60.0` in the run directory name and checkpoint paths is misleading — it was generated by the `--lambda 60` argument parsing, but the actual training never used this value.

## Fix Applied

Removed the lambda restore line from `load_checkpoint`:

```python
# Removed: self.lambda_val = ckpt.get("lambda_val", self.lambda_val)
```

CLI `--lambda` is now the single source of truth for the RD weight.
