The Mismatch

  • 82% of internet traffic is video. The fastest-growing consumer is a neural network.
  • H.264/5 optimizes for pixel fidelity — PSNR, smooth gradients, artifact-free motion.
  • Machine perception doesn't need beautiful pixels. It needs preserved semantic structure at minimum bitrate.

The Decoding Tax

  • Every detection pipeline decodes full-resolution frames → discards semantics → re-infers from pixels.
  • Storage and bandwidth optimized for the wrong objective.
  • Motion vectors consume 30–40% of bitstream at low rates — wasted on machines.

Structural market failure: Every camera compressing with H.265 today is optimizing for the wrong consumer. This requires a new codec stack, not a software update.

04 // SOLUTION

Two products. One infrastructure layer.

LeWM-VC

Codec built for machine perception. Compresses to a semantic latent representation rather than pixels.

05 // TECHNOLOGY

The insight in one slide.

Traditional Codecs

Predict frames in pixel space using hand-crafted motion vectors. Optimized for visual reconstruction.

↓

H.264/5: ME/MC + DCT + CABAC • PSNR optimized

LeWM-VC

Predict frames in latent semantic space using a learned transformer. Optimized for what downstream models need.

↓

ViT Encoder → JEPA Predictor → Residual Coding • No motion vectors

Result: Smaller files. Better machine perception. No motion vector overhead. The JEPA predictor models scene dynamics through self-attention — object permanence, occlusion, coherent motion — without the rigidity of block-based estimation.

The evaluation standard is the moat.

BD-rate is a one-page ITU document from 2001 that became the mandatory metric for every codec paper for 25 years. Its author did not invent H.264. He defined how to measure it.
